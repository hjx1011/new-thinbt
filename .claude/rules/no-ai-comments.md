# 注释规范

## 禁止的注释
- "This function does X" — 函数名已经说明的事不需要重复
- "TODO: optimize later" — 要么现在做，要么不提
- 逐行解释代码在干嘛 — 代码本身应该自解释

## 好的注释
- **为什么这么做**，不是做了什么
- 非显而易见的边界条件（如 "chunk 不可分割，必须完整下载"）
- 算法选择的理由（如 "用 fastCDC 而非固定分块，因为..."）
- 已知问题或限制（如 "Windows 下不支持 xxx"）

## 原则
注释解释的是设计决策和业务逻辑，不是代码语法。如果代码需要注释才能看懂，先考虑重命名变量和拆分函数。
