​局域网高并发大文件分发与增量同步系统设计文档
​1. 系统概述与需求边界
​本系统旨在解决局域网内（1 台服务端节点至 60 台客户端节点）超大文件（50GB）的极速分发与高频增量更新问题。
​核心挑战： 突破服务端单点网络 I/O 瓶颈、解决异构网络（千兆/百兆混杂）短板效应、避免底层 NTFS 文件系统碎片化。
​核心目标： 分发时间趋近于局域网物理带宽极限；实现按需拉取变更块的极速增量同步。
​2. 整体架构设计
​系统采用 轻量级中心化调度 + 纯去中心化 P2P 数据传输 的混合架构。
​Tracker 调度中心： 部署于主节点（老师机），仅负责网络状态同步、服务注册与节点 IP 交换，绝对不参与实际数据流转。
​P2P 数据平面： 60 台节点互相建立网状并发连接。借鉴高并发分布式后端的连接管理模型，通过高效的 I/O 多路复用机制（如 epoll）处理海量对等节点（Peer）的数据交换。
​3. 核心分发策略与算法
​3.1 节点发现与高可用路由
​主干发现： 客户端启动后，向 Tracker 服务器注册自身状态并获取局域网内的 Peer 列表。
​容灾兜底（PEX）： 客户端在后台维持 Gossip 协议进行节点交换（Peer Exchange）。一旦 Tracker 宕机，已连接的节点仍可通过相互询问获取新节点 IP，确保网络调度的鲁棒性。
​3.2 异构网络带宽调度（千兆/百兆融合）
​分层拓扑： 协议层引入带宽感知。千兆节点互联形成高速骨干网（Backbone），优先互相全速对拷大尺寸数据块（如 8MB）。
​边缘降级： 百兆节点被降级为边缘节点，主要负责吸收网络中已经高度冗余的数据块，避免其占用千兆节点的核心并发连接数。
​3.3 块选择与调度算法
​最稀缺优先（Rarest First）： 核心算法策略。节点通过交换位图（Bitfield）统计全网数据块分布，贪心拉取副本数最少的块，迅速拉升全网容灾率。
​终局模式（Endgame）： 针对最后 1% 的长尾下载阶段，节点向所有持有该碎块的 Peer 并发请求，最先返回数据的连接获胜并立即向其余节点发送 Cancel 指令，消除慢速节点的木桶效应。
​4. 底层 I/O 与并发控制（C++ 后端实现细节）
​针对 50GB 级别的数据吞吐，系统必须在内存、磁盘和网络协议栈之间实现极致的性能榨取。
​4.1 磁盘与内存管理优化
​连续空间预分配： 彻底解决 NTFS 乱序写入导致的灾难性碎片问题。在接收首字节前，调用底层 API 直接在磁盘上划定 50GB 连续物理扇区。
​内存映射（mmap）： 摒弃传统的用户态 read/write。使用 mmap 将这 50GB 的物理文件直接映射到进程的虚拟内存空间中，将应用层的乱序网络块接收转化为对内存数组的直接覆盖，由操作系统的 Page Cache 负责将脏页批量平滑地刷入磁盘。
​零拷贝推流（Zero-copy）： 当本节点向其他 Peer 分发已落盘的数据块时，直接使用 sendfile 系统调用。数据在内核态直接从 Page Cache 拷贝至网卡 Socket 缓冲区，将 CPU 的数据搬运开销降至零。
​4.2 高并发状态机与线程安全
​连接上下文管理： 利用 shared_ptr 管理数十个 Peer 连接的生命周期，避免在复杂的异步读写回调中发生野指针或重复析构。
​状态同步与无锁化： shared_ptr 的引用计数本身是线程安全的，但其管理的 Peer 状态对象并不是。为了避免网络 I/O 线程和后台校验线程发生数据竞争，系统采用**无锁队列（Lock-free Queue）**进行事件派发，将状态更新操作严格串行化。
​进程模型： 核心网络框架必要时可通过 fork 创建独立的 Worker 子进程来处理高密度的哈希计算任务，隔离网络事件循环与 CPU 密集型操作。
​5. 增量更新与一致性校验
​分块滚动哈希： 文件被切分为固定大小的 Block。主节点修改文件后，重新生成包含所有 Block 的 SHA-256 哈希列表（种子文件）。
​本地快速比对： 客户端导入新种子并指向本地原 50GB 文件目录。系统通过 mmap 以几百 MB/s 的速度扫描本地文件，与新种子进行哈希碰撞。
​差异化拉取： 仅将被修改导致哈希失效的块标记为“缺失”，随后客户端仅向 P2P 网络请求这极少量的变更块，完美实现毫秒级的增量下载。
